# UI_Guidelines.md

# Tennis Courts Academy

## Guide d'interface du tutoriel

Version 1.0

------------------------------------------------------------------------

# Objectif

Le tutoriel s'intègre entièrement à l'interface actuelle de Tennis
Courts Academy.

Aucun écran parallèle ne doit être créé.

Le joueur doit avoir l'impression de jouer au jeu normal avec un
accompagnement.

------------------------------------------------------------------------

# Plateau de jeu

Le plateau actuel est conservé.

Ne jamais déplacer :

-   les joueurs
-   les mains de cartes
-   la carte centrale
-   les boutons
-   les compteurs

Le tutoriel ne modifie pas la disposition existante.

------------------------------------------------------------------------

# Zone de dialogue

Ajouter une zone de dialogue fixe en bas de l'écran.

Caractéristiques :

-   pleine largeur
-   lisible
-   contraste élevé
-   marges confortables
-   responsive

Cette zone est réservée aux dialogues de Coach Ju.

------------------------------------------------------------------------

# Portrait de Coach Ju

Toujours visible pendant un dialogue.

Position :

-   à gauche de la zone de dialogue.

Contraintes :

-   le texte ne passe jamais sous le portrait ;
-   conserver cette logique sur mobile.

------------------------------------------------------------------------

# Cartes agrandies

Lorsqu'une carte est expliquée :

-   afficher une version agrandie ;
-   conserver le plateau visible en arrière-plan.

La carte agrandie doit pouvoir mettre en évidence :

-   coût
-   puissance
-   précision
-   placement
-   effet
-   zone boost

------------------------------------------------------------------------

# Mises en évidence

Le moteur doit pouvoir surligner :

-   une carte
-   un bouton
-   une jauge
-   la carte centrale
-   une zone précise d'une carte

Le surlignage doit disparaître dès que l'étape est validée.

------------------------------------------------------------------------

# Flèches

Les flèches servent uniquement à guider une action.

Exemples :

-   sélectionner une carte
-   cliquer sur Jouer
-   cliquer sur Passer
-   cliquer sur Boost

Ne jamais afficher plusieurs flèches si une seule suffit.

------------------------------------------------------------------------

# Messages

Coach Ju parle avec des phrases courtes.

Le texte doit rester facilement lisible.

Éviter les longs paragraphes.

------------------------------------------------------------------------

# Gestion des erreurs

En cas d'action incorrecte :

-   afficher une explication courte ;
-   conserver l'étape active ;
-   permettre un nouvel essai immédiatement.

------------------------------------------------------------------------

# Responsive

PC : - conserver le plateau actuel.

Tablette : - conserver la même logique de lecture.

Mobile : - aucune rupture de disposition. - priorité à la lisibilité. -
éviter tout recouvrement du portrait et du texte.

------------------------------------------------------------------------

# Principes UX

Toujours montrer avant d'expliquer.

Toujours expliquer avant de demander une action.

Toujours confirmer une bonne action avant de passer à l'étape suivante.

Le joueur ne doit jamais hésiter sur l'action attendue.
